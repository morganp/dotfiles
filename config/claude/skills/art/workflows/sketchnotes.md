# Sketchnotes Workflow

**Hand-drawn visual notes that capture key ideas, themes, and relationships from content.**

Creates **AUTHENTIC SKETCHNOTE SUMMARIES** -- visual notes combining hand-lettering, simple icons, containers, and connectors to distill content into a scannable visual format.

---

## Purpose

Sketchnotes are visual summaries that capture the essence of content through a combination of text, drawings, and structure. Unlike diagrams (which show structure) or infographics (which present data), sketchnotes show **ideas captured visually** as if sketched by hand during a talk or meeting.

**Use this workflow for:**
- Meeting or talk summaries
- Concept overviews
- Book/article visual summaries
- Event recaps
- Training session captures
- Newsletter visual summaries

---

## Visual Aesthetic: Authentic Hand-Drawn Notes

**Think:** Visual notes sketched by hand during a live talk with markers on a large pad

### Core Characteristics
1. **Hand-lettered text** -- Multiple sizes showing hierarchy, not typed
2. **Simple icons** -- 5-15 line drawings, not detailed illustrations
3. **Containers** -- Banners, bubbles, boxes grouping related content
4. **Connectors** -- Arrows and lines showing flow and relationships
5. **Imperfect lines** -- Deliberate wobble, not ruler-straight
6. **Generous negative space** -- At least 25-30% empty

---

## The Five Building Blocks

Every sketchnote is composed of five elements in combination.

### 1. Text (Hand-Lettering)
| Style | When to Use |
|-------|-------------|
| Bold block letters | Titles, main headings |
| ALL CAPS sans-serif | Section headers |
| Script/cursive | Quotes, attributions |
| Small neat print | Body text, annotations |
| Bubble letters | Fun emphasis on key terms |

**Rule:** Maximum 3 text styles per sketchnote.

### 2. Basic Shapes
Circles, rectangles, triangles, lines, dots, stars, arrows. All drawn with slightly imperfect/wobbly lines.

### 3. Containers
| Container | Use For |
|-----------|---------|
| Ribbon banner | Titles, section headers |
| Speech bubble | Quotes, dialogue |
| Thought bubble | Ideas, internal reflections |
| Rounded rectangle | General grouping |
| Starburst | Key stats, surprises |
| Bracket | Grouping related items |

**Rule:** 2-4 container types per sketchnote (variety without chaos).

### 4. Connectors
Arrows (straight, curved), dotted lines, numbered paths, branches. Hand-drawn arrowheads, not geometric.

### 5. Icons
Simple pictorial representations (5-15 lines each): lightbulb, gear, people, target, chart, clock, book, rocket, magnifying glass, megaphone, flag, trophy, key, bridge, tree, star.

**Rule:** 4-8 icons per single-page sketchnote.

## Layout Patterns

| Layout | Best For | Description |
|--------|----------|-------------|
| **Linear** | Chronological, agendas | Top-to-bottom flow with sections |
| **Radial** | Brainstorms, themes | Central idea with topics radiating out |
| **Path** | Processes, journeys | Winding road connecting stops |
| **Modular** | Parallel topics | Distinct sections arranged as panels |
| **Columns** | Multi-speaker, comparisons | Side-by-side vertical sections |
| **Skyscraper** | Dense agenda content | Horizontal dividers separating vertical sections |
| **Popcorn** | Unstructured, creative | Organic placement scattered across page |

## Color System

### Structure
```
Charcoal        #2D2D2D   (all drawing, text, icons, lines)
```

### Emphasis
```
Deep Teal       #1A6B6B   (key themes, main containers, important connectors)
Burnt Orange    #C85A2A   (action items, emphasis, alerts)
```

### Background
```
Warm Cream      #F7F4EA   (primary background)
Light Teal      #E6F3F3   (subtle container fills, optional)
```

### Color Strategy
- All drawing in charcoal ink
- Teal accent on primary themes and key containers
- Burnt orange on action items and calls to attention
- Marker-style highlight blobs behind key phrases
- Maximum 3 accent colors total

## MANDATORY WORKFLOW STEPS

### Step 1: Define the Content

**Extract the key information to visualize:**

1. **What is this about?** [Topic/title in 5-8 words]
2. **How many key themes?** [3-7 sections]
3. **What layout fits?** [Use layout selection guide below]
4. **Key stats or numbers?** [0-3 standout figures]
5. **Notable quotes?** [0-2 attributed quotes]
6. **Action items?** [0-5 next steps]
7. **Visual metaphors?** [1-3 concepts that map to images]

**Layout Selection Guide:**
```
Is it chronological?
  YES -> Is it a journey? -> PATH
         Is it agenda items? -> LINEAR
  NO  -> Is there one central theme? -> RADIAL
         Are there parallel topics? -> COLUMNS or MODULAR
         Is it dense with many sections? -> SKYSCRAPER
         Is it unstructured/creative? -> POPCORN
```

**Output:**
```
TITLE: [5-8 word title]
LAYOUT: [Linear / Radial / Path / Modular / Columns / Skyscraper / Popcorn]

SECTIONS:
1. [Section heading] -- [2-3 key points]
   Icon: [simple icon name]
   Container: [banner / bubble / rectangle / starburst]
   Importance: [primary / secondary / tertiary]

2. [Section heading] -- [2-3 key points]
   Icon: [icon name]
   ...

KEY STATS: [stat 1], [stat 2]
QUOTES: "[quote]" -- Speaker
ACTION ITEMS: [action 1], [action 2]
VISUAL METAPHORS: [metaphor 1], [metaphor 2]
MOOD: [energetic / serious / collaborative / creative / urgent]
```

---

### Step 2: Plan Visual Hierarchy

**Determine size and emphasis for each element:**

| Element | Size | Treatment |
|---------|------|-----------|
| Title | Largest (3-5x body text) | Bold block letters in ribbon banner |
| Primary sections | Medium-large | UPPERCASE header, bold container, teal accent |
| Secondary sections | Medium | Standard container, regular weight |
| Details/annotations | Small | Neat print, margin placement |
| Key stats | Medium-large | Starburst container, bold |
| Action items | Medium | Bordered box, burnt orange accent |

**Spatial Placement:**
- Title: Top center (most prominent)
- Key themes: Upper half or center
- Action items: Bottom section
- Annotations: Margins
- Metadata (date, author): Corners

---

### Step 4: Determine Aspect Ratio

| Sketchnote Type | Aspect Ratio | Reasoning |
|-----------------|--------------|-----------|
| Standard summary | 3:4 | Classic sketchnote portrait format |
| Presentation/widescreen | 16:9 | Horizontal flow, good for sharing |
| Social media share | 1:1 | Square format for social platforms |
| Detailed multi-section | 9:16 | Tall format for many sections |
| Summary card (compact) | 4:3 | Compact landscape |

**Default: 3:4** -- Classic sketchnote proportions

---

### Step 5: Execute Generation

```bash
bun run ~/.claude/skills/art/tools/generate-image.ts \
  --prompt "[YOUR PROMPT]" \
  --size 2K \
  --aspect-ratio 3:4 \
  --output /path/to/sketchnote.png
```

**Model:** Nano Banana 2 (default) (best hand-drawn aesthetic and text rendering for sketchnotes)

---

### Step 6: Validation (MANDATORY)

#### Must Have
- [ ] **Hand-drawn quality** -- Lines are wobbly/imperfect, not ruler-straight
- [ ] **Hand-lettered text** -- Text looks written, not typed or printed
- [ ] **Visual hierarchy** -- Title largest, clear size progression
- [ ] **Containers** -- At least 2 container types visible
- [ ] **Icons** -- Simple hand-drawn icons as visual anchors
- [ ] **Negative space** -- At least 25% of page is empty
- [ ] **Readable text** -- All text is legible
- [ ] **Accent colors** -- Teal and/or orange accents visible

#### Must NOT Have
- [ ] Ruler-straight lines or perfect geometry
- [ ] Typed/printed looking text
- [ ] Corporate diagram aesthetic
- [ ] Drop shadows or gradients
- [ ] Clip art style icons
- [ ] Wall-of-text sections
- [ ] Leaked prompt instructions visible in image
- [ ] Rainbow/excessive colors

#### If Validation Fails

| Problem | Fix |
|---------|-----|
| Too polished/digital | Add "rough imperfect hand-drawn strokes, wobbly lines like drawn on whiteboard" |
| Text illegible | Reduce text quantity, increase text size, simplify wording |
| No visual hierarchy | Specify title 3-5x larger, vary section sizes by importance |
| Too cluttered | Reduce to 4-6 elements max, increase negative space directive |
| Looks like infographic | Strengthen "NOT an infographic" negative prompt, add "sketched during live talk" |
| Prompt leakage | Restructure prompt to keep layout instructions separate from content |
| Wrong layout | Be more explicit about spatial arrangement in prompt |

## Density Guidelines

| Content Length | Max Visual Elements | Coverage |
|---------------|-------------------|----------|
| Short (< 2000 words) | 4-6 | 60-80% of themes |
| Medium (2000-5000 words) | 6-8 | 40-60% of themes |
| Long (5000-10000 words) | 8-10 (absolute max) | 20-40% of themes |

**Rule:** Never exceed 10 visual elements on a single page. For longer content, suggest multi-image series.

## Example Use Cases

### Example 1: Weekly Team Meeting Summary
- Layout: Linear (agenda-driven)
- Sections: Sprint update, blockers, wins, action items
- Icons: gear, lightning bolt, trophy, checkbox
- Stats: "87% sprint completion"

### Example 2: Conference Talk Summary
- Layout: Radial (central theme with key takeaways)
- Sections: Main thesis, 3-4 supporting arguments, call to action
- Icons: megaphone, lightbulb, rocket, target

- Layout: Path (journey through key ideas)
- Sections: Major concepts as stops along the path
- Icons: book, key, bridge, growing plant

## Quick Reference

**The Formula:**
```
1. Define content (themes, stats, quotes, actions, metaphors)
2. Select layout (linear, radial, path, modular, columns, skyscraper, popcorn)
3. Plan visual hierarchy (title > primary > secondary > details)
4. Construct prompt with visual vocabulary
5. Choose aspect ratio (usually 3:4)
6. Generate image
7. Validate authenticity and readability
```

**Element Count Guidelines:**
- Title: 1 (always)
- Sections: 3-7
- Icons: 4-8
- Stats: 0-3
- Quotes: 0-2
- Action items: 0-5
- Total visual elements: Never exceed 10 on one page

---

**The workflow: Define -> Plan -> Construct -> Generate -> Validate -> Complete**
