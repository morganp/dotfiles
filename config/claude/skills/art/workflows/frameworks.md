# Frameworks Workflow

**Hand-drawn 2x2 matrices, Venn diagrams, and mental model visualizations.**

Creates **STRATEGIC THINKING TOOLS** — visual frameworks that help audiences think through problems using established mental models.

---

## Purpose

Frameworks externalize mental models. They help audiences understand relationships, trade-offs, and positioning. Unlike taxonomies (which categorize) or comparisons (which contrast), frameworks reveal **underlying structure** of thinking.

**Use this workflow for:**
- 2x2 matrices (position mapping)
- Venn diagrams (overlaps)
- Quadrant analysis
- Mental model visualization
- Strategic positioning maps
- Trade-off illustrations

---

## Visual Aesthetic: Whiteboard Strategy

**Think:** Consultant's whiteboard during a strategy session

### Core Characteristics
1. **Clear axes/dimensions** — What each axis measures
2. **Hand-drawn quality** — Wobbly lines, sketchy quadrants
3. **Strategic positioning** — Items placed meaningfully
4. **Labeled regions** — Each quadrant/section named
5. **Visual emphasis** — Key positions highlighted

---

## Framework Types

### 2x2 Matrix
- Two axes creating four quadrants
- Items positioned based on two dimensions
- Each quadrant labeled with meaning
- **Example:** Eisenhower Matrix (Urgent/Important)

### Venn Diagram
- 2-3 overlapping circles
- Shows intersections and unique areas
- Labels for each region
- **Example:** Skills overlap diagram

### Spectrum/Continuum
- Single axis with positions along it
- Shows range from one extreme to another
- **Example:** Build vs Buy spectrum

### Pyramid/Hierarchy
- Stacked levels showing progression
- Base to apex structure
- **Example:** Maslow's hierarchy

## Color System

### Structure
```
Charcoal        #2D2D2D   (axes, lines, text)
Dark Gray       #4A4A4A   (secondary lines, labels)
Light Gray      #E5E7EB   (grid lines, subtle divisions)
```

### Emphasis
```
Deep Teal       #1A6B6B   (key quadrant or ideal position)
Burnt Orange    #C85A2A   (secondary emphasis, contrast)
Bright Teal     #2B9B9B   (lighter fill variation)
```

### Background
```
Warm Cream      #F7F4EA   (primary)
Pure White      #FFFFFF   (alternative)
```

### Color Strategy
- Highlight the "ideal" quadrant or key intersection
- Use subtle fills to differentiate regions
- Keep most structure in charcoal
- Color draws attention to the insight

## MANDATORY WORKFLOW STEPS

### Step 1: Define Framework Structure

**Identify the mental model:**

1. **What type of framework?** [2x2 / Venn / Spectrum / Pyramid]
2. **What dimensions/axes?** [Name each axis]
3. **What regions exist?** [Name each quadrant/section]
4. **What's positioned where?** [Items to plot]
5. **What's the key insight?** [Where should attention go]

**Output for 2x2:**
```
FRAMEWORK: 2x2 Matrix
TITLE: [Framework name]

X-AXIS: [Dimension] — [Low label] ←→ [High label]
Y-AXIS: [Dimension] — [Low label] ←→ [High label]

QUADRANTS:
- Top-Right: [Name] — [Description]
- Top-Left: [Name] — [Description]
- Bottom-Right: [Name] — [Description]
- Bottom-Left: [Name] — [Description]

ITEMS TO PLOT:
- [Item 1]: [Position]
- [Item 2]: [Position]
...

KEY INSIGHT: [Which quadrant/position to highlight]
```

**Output for Venn:**
```
FRAMEWORK: Venn Diagram
TITLE: [Framework name]

CIRCLES:
- Circle A: [Label]
- Circle B: [Label]
- Circle C: [Label] (optional)

REGIONS:
- A only: [What lives here]
- B only: [What lives here]
- A ∩ B: [What lives in overlap]
- A ∩ B ∩ C: [Center overlap] (if 3 circles)

KEY INSIGHT: [Which region to highlight]
```

---

### Step 2: Design Layout

**Plan the visual structure:**

1. **Framework placement:** Center with room for labels
2. **Label positions:** Clear, readable, near their regions
3. **Item placement:** Where things plot on the framework
4. **Emphasis:** Which region gets color highlight

**Output:**
```
LAYOUT:
- Framework: [Centered / offset]
- Title: [Position]
- Axis labels: [Positions]
- Quadrant labels: [Inside / outside quadrants]
- Items: [How they're shown — dots, icons, text]
- Highlight: [Which region gets teal/orange]
```

---

### Step 3: Construct Prompt

### Prompt Template (2x2 Matrix)

```
Hand-drawn 2x2 matrix framework in Excalidraw sketch style on warm cream background (#F7F4EA).

STYLE REFERENCE: Strategy consulting whiteboard, hand-drawn quadrant analysis, business framework sketch

BACKGROUND: Warm Cream #F7F4EA — clean, flat

AESTHETIC:
- Hand-drawn axes (wobbly lines, not ruler-straight)
- Sketchy quadrant divisions
- Variable stroke weight
- Whiteboard/consulting aesthetic
- Strategic but approachable

FRAMEWORK: [Title]

AXES:
- X-AXIS: "[Dimension]"
  * Left end: "[Low label]"
  * Right end: "[High label]"
- Y-AXIS: "[Dimension]"
  * Bottom: "[Low label]"
  * Top: "[High label]"

QUADRANTS:

TOP-RIGHT: "[Quadrant Name]"
- Description: [Brief meaning]
- Style: [Teal fill #1A6B6B if this is key quadrant]

TOP-LEFT: "[Quadrant Name]"
- Description: [Brief meaning]
- Style: [Fill treatment]

BOTTOM-RIGHT: "[Quadrant Name]"
- Description: [Brief meaning]
- Style: [Fill treatment]

BOTTOM-LEFT: "[Quadrant Name]"
- Description: [Brief meaning]
- Style: [Fill treatment]

ITEMS PLOTTED: (if applicable)
- [Item 1]: [Position in matrix]
- [Item 2]: [Position in matrix]

TYPOGRAPHY:
- Title: Bold hand-lettered, charcoal #2D2D2D
- Axis labels: Medium, at axis ends
- Quadrant names: Inside or near each quadrant
- All text: Charcoal #2D2D2D

COLOR USAGE:
- Axes and structure: Charcoal #2D2D2D
- Key quadrant fill: Subtle Teal (#1A6B6B at 20% opacity)
- Contrast quadrant: Subtle Orange (#C85A2A at 20% opacity)
- Other quadrants: Light gray or unfilled

LAYOUT:
- Matrix centered
- Clear axis labels at each end
- Quadrant labels visible
- 30-40% negative space
- Professional but hand-drawn

CRITICAL REQUIREMENTS:
- Hand-drawn quality (NOT digital perfect lines)
- Clear axis labels and quadrant names
- Strategic emphasis on key quadrant
- Readable and scannable
- Consulting-quality but approachable
```

### Prompt Template (Venn Diagram)

```
Hand-drawn Venn diagram in Excalidraw sketch style on warm cream background (#F7F4EA).

[Similar structure, adapted for overlapping circles]

CIRCLES:
- Circle A: "[Label]" — [Position and color]
- Circle B: "[Label]" — [Position and color]
- Circle C: "[Label]" — [Position and color] (if 3)

OVERLAP REGIONS:
- A ∩ B: "[What lives here]"
- A ∩ B ∩ C: "[Center — key insight]" — Teal highlight

[Continue with similar structure...]
```

### Step 4: Determine Aspect Ratio

| Framework Type | Aspect Ratio | Reasoning |
|----------------|--------------|-----------|
| 2x2 Matrix | 1:1 | Square for balanced quadrants |
| Wide 2x2 | 16:9 | Room for extended labels |
| 2-circle Venn | 16:9 | Horizontal circles |
| 3-circle Venn | 1:1 | Balanced triangle arrangement |
| Spectrum | 21:9 | Wide for linear range |
| Pyramid | 9:16 or 4:5 | Tall for vertical hierarchy |

**Default: 1:1** — Most frameworks work square

---

### Step 5: Execute Generation

```bash
bun run ~/.claude/skills/art/tools/generate-image.ts \
  --prompt "[YOUR PROMPT]" \
  --size 2K \
  --aspect-ratio 1:1 \
  --output /path/to/framework.png
```

**Model:** Nano Banana 2 (default). Add `--thinking high` for precise quadrant positioning.

---

### Step 6: Validation (MANDATORY)

#### Must Have
- [ ] **Clear structure** — Axes/circles/regions obvious
- [ ] **Readable labels** — All text legible
- [ ] **Hand-drawn quality** — Wobbly lines, sketchy feel
- [ ] **Strategic emphasis** — Key region highlighted
- [ ] **Meaningful regions** — Each quadrant/area labeled

#### Must NOT Have
- [ ] Perfect geometric shapes
- [ ] Missing axis labels
- [ ] Unlabeled quadrants/regions
- [ ] No visual hierarchy (everything same weight)
- [ ] Corporate polished look

#### If Validation Fails

| Problem | Fix |
|---------|-----|
| Too perfect | "Hand-drawn whiteboard sketch, wobbly axes, organic lines" |
| Labels unclear | Increase label size, position outside busy areas |
| No hierarchy | Strengthen teal fill on key quadrant |
| Too complex | Simplify to core insight, reduce text |

## Quick Reference

**The Formula:**
```
1. Define framework structure (type, axes, regions)
2. Design layout (positions, labels, emphasis)
3. Construct prompt with strategic coloring
4. Choose aspect ratio (usually 1:1)
5. Generate image (add --thinking high for complex layouts)
6. Validate structure clarity and emphasis
```

**Framework Quick Decision:**
```
What are you showing?
├─ Two dimensions, four options → 2x2 Matrix
├─ Overlap between concepts → Venn Diagram
├─ Range from A to B → Spectrum
└─ Levels/hierarchy → Pyramid
```

**The workflow: Define → Design → Construct → Generate → Validate → Complete**
