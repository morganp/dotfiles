# Terminal Table Formatting

## Purpose

Use this skill to produce readable terminal tables. The table must be generated by the bundled formatter script, not hand-aligned.

The formatter computes visible column widths from the data, strips ANSI escape sequences when measuring widths, pads cells with spaces, and emits a fixed-width ASCII table.

## When To Use

Use this skill when output includes:
- comparison tables
- check/result/status summaries
- code review finding tables
- spec/config/RTL comparison tables
- command output summaries that are easier to read as rows and columns
- any terminal-facing table with more than trivial content

Do not use this skill for tables intended to be edited into markdown documents, READMEs, specs, or source files unless the user explicitly wants an ASCII terminal table.

## Required Workflow

1. Build the table data as JSON with `headers` and `rows`.
2. Run `scripts/format_table.py` with that JSON on stdin.
3. Paste the generated table exactly.
4. Do not hand-pad or manually repair table alignment.

Use the absolute path when calling from another working directory:

```bash
python3 /home/morpri01/.codex/skills/terminal-table-formatting/scripts/format_table.py <<'JSON'
{
  "headers": ["Status", "Check", "Result"],
  "rows": [
    ["PASS", "Protocol", "YAML matches spec."],
    ["MISMATCH", "Width", "Spec says 8b; YAML has 9b."]
  ]
}
JSON
```

For color, prefer semantic cell objects instead of raw ANSI escape sequences in the JSON:

```bash
python3 /home/morpri01/.codex/skills/terminal-table-formatting/scripts/format_table.py <<'JSON'
{
  "headers": ["Status", "Check", "Result"],
  "rows": [
    [{"text": "PASS", "color": "green"}, "Protocol", "YAML matches spec."],
    [{"text": "MISMATCH", "color": "red"}, "Width", "Spec says 8b; YAML has 9b."],
    [{"text": "CHECK", "color": "yellow"}, "Resource plane", "Needs design-owner confirmation."]
  ]
}
JSON
```

Use `--no-color` to ignore semantic color fields and emit plain text. ANSI escape sequences already present in input cells are still ignored when calculating visible string width.

The output shape is:

```text
+----------+----------+------------------------------+
| Status   | Check    | Result                       |
+----------+----------+------------------------------+
| PASS     | Protocol | YAML matches spec.           |
| MISMATCH | Width    | Spec says 8b; YAML has 9b.   |
+----------+----------+------------------------------+
```

## Rules

- Always use the script instead of manually aligning terminal tables.
- Keep table cells single-line. If content is too long, summarize it before passing it to the script.
- Use plain text cells by default. If color improves readability, use semantic cell objects with `text` and `color`.
- Do not put raw ANSI escape sequences in JSON unless there is a specific reason. The script supports them for width calculations, but semantic color fields are easier to read and disable.
- Prefer concise column names.
- If a table becomes too wide for the terminal, shorten cell text rather than wrapping by hand.
- For markdown-native output, use normal markdown tables only when terminal alignment is not the priority.
